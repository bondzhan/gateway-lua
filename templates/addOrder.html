行云订单API v3.1

{(./nav.html)}

增加销售订单

简要描述:

  • 接口地址：***.**.**.**/api/service/business
  • 支持格式：JSON
  • 请求方式：HTTP POST
请求参数说明
名称 类型 必填 说明
opcode String 是 操作码，固定值: add_order
merchant_order_no String 是 订单标识Id(一般是对接方的订单号)
pay_type String 是 付款方式:1余额支付,2信用支付
merchant_id String 是 商户id
accept_name String 是 收货人姓名
card_id String 是 身份证号码
post_code String 是 邮政编码(6位长度)
telphone String 否 联系电话
mobile String 是 手机
province String 是 所属区域：省
city String 是 所属区域：市
area String 是 所属区域：区
address String 是 收货地址
card_url_front String 否 身份证正面URL部分直邮必填
card_url_back String 否 身份证背面URL部分直邮必填
message String 否 订单留言
items sku_id String 是 商品编码(SKU_ID)
quantity int 是 订购数量
sign String 是 MD5加密串
sign_type String 是 加密方式:MD5(大写)
返回参数说明
名称 类型 说明
ret_code String 业务结果返回
ret_msg String 业务结果描述返回
sign String MD5加密串
sign_type String 加密方式:MD5(大写)
status String 支付订单状态1待支付 2支付成功 3已取消
order_no String 支付订单流水(业务成功则有返回)
merchant_order_no String 外部订单标识(您传入的订单号)
items goods_order String 商品订单(拆分的订单)
sku_id String sku编码
freight_amount String 运费
buy_num String 购买数量
price String 单价
order_status String 订单状态: 1.待付款 2.待确认 3.通关中 4.待推送 5.待发货 6.待收货 7.已完成 8.后台管理员取消该笔商品订单(异常订单) 9.发货失败 10过期或用户取消订单
total_all_amount String 订单总额(业务成功则有返回)
real_pay_amount String 实际支付金额
total_freight String 运费(业务成功则有返回)
